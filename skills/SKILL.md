# Moegirl OC 分析工具

## 项目定位

这是一个用于给原创 OC（原创角色）编写**萌娘百科风格** Markdown 文件的工具。核心思路是：爬取萌娘百科的角色属性数据，构建「角色 × 属性」二部图统计模型，然后用来分析新角色并生成符合萌娘百科语境的文档。

**用户不使用源码**，用户只关心：提交个人OC信息（可能是文本，图片或复合型资源） → 得到一份 md 文件。

---

## 分析工作流

> **核心原则**：严格遵循四步顺序执行，每一步完成后才能进入下一步。第四步是最终质量门禁——有任何属性在参考.txt中找不到对应条目，必须回头修正第三步的输出，绝不将未经核实的属性带入最终文档。

---

### 第一步：读取规范格式文件

**目的**：学习萌娘百科风格文档的完整结构与行文规范。

读取 `data/加里波.md`（本项目的规范终稿），从中提取以下结构要素：

- **标题格式**：`= 角色名 =`，角色名加粗 `'''角色名'''`
- **简介段落**：一句话交代角色出处与定位
- **基础信息表格**：姓名、性别、年龄、所在地、种族、家人/关系
- **萌属性一览表**：按大类→小类（`【】`标签）→属性表（三列：萌属性 | 强度 | 出典）分层
- **详细说明**：2-5个核心萌点，每个用 `=== 标题【标签】 ===` 结构，配以 `>` 块引用原文
- **经典语录**：分类列举，`> 「台词」` 格式
- **形象总结**：结尾的总结段落
- **落款格式**：`*本文档基于《作品名》系列创作。*` + `*最后更新：YYYY年MM月DD日*`

**输出**：在脑海中建立完整的格式模板，明确各部分的层级关系和行文风格。

---

### 第二步：读取用户提交的所有信息

**目的**：完整获取角色的原始设定数据。

读取用户提供的所有提交内容（文本描述、图片、混合资源等），从中提取：

| 信息类别 | 提取要点 |
|---|---|
| 角色名 | 必填，确认是否有别名/作品内称呼 |
| 性别 | 必填，结合 male_attr.json / female_attr.json 推断 |
| 作品/世界观 | 作品名、作者、设定集来源 |
| 外貌特征 | 发型、发色、瞳色、体型、肤色等 |
| 性格特征 | 明确描述的性格关键词（傲娇、元气、腹黑等） |
| 人际关系 | 家人、朋友、恋人、师徒等 |
| 特殊能力/身份 | 职业、种族、特殊能力 |
| 经典台词/设定原文 | 用于出典引用（优先提取具体文本） |
| 服装/饰品 | 具体服装描述 |

**注意**：如果用户提交的是图片，需要识别其中的视觉特征（发型发色、服装风格等）并转化为文字描述。

---

### 第三步：属性匹配与第一版文档生成

**目的**：从用户描述中匹配萌属性，参照 attr_desc.json 书写描述，产出第一版 md 文件。

#### 3.1 属性匹配

将第二步提取的关键词与以下数据源匹配：

| 数据源 | 用途 |
|---|---|
| `data/attr_index.json` | 精确匹配已知属性 |
| `data/hair_color_attr.json` | 发色匹配 |
| `data/eye_color_attr.json` | 瞳色匹配 |
| `data/male_attr.json` / `data/female_attr.json` | 性别相关属性 |

**匹配策略**：
- **精确匹配**：直接命中 `attr_index.json` 中的条目
- **模糊匹配**：
  - "银白色头发" → `"银发"`
  - "红蓝异瞳" → `"异色瞳"`
  - "身材高挑" → `"高挑"`（若存在）或体型相关属性
  - "元气少女" → `"元气"` + `"少女"`（成长阶段）
- **互信息排序**：当多个属性都可能时，优先选择共现频率更高的组合

#### 3.2 属性归类

> **重要**：此处的归类**仅作为草稿**，所有属性在第四步必须经过参考.txt的最终核验。

参照 `data/参考.txt` 的分类体系，对每个匹配到的属性确定其所属：

1. **读取 `data/参考.txt`**，按行解析，识别 `【】` 中的小类标签
2. 查找每个属性出现在哪个 `【】` 分组下
3. 若属性不在 `参考.txt` 中，先暂存，标记为【待核验-自创分类】，等待第四步处理

**大类对应关系**（来自参考.txt）：

| 大类 | 小类标签示例 |
|---|---|
| 性格类 | 【浅层性格】【深层性格】【性格组合】 |
| 外表年龄段及性别特征类 | 【性别总论】【成长阶段】 |
| 相貌、体型特征类 | 【发型】【发色】【瞳色】【眼型】【体型】【皮肤】【口部】【胸部】等 |
| 服装、饰品、着装状态类 | 【服装】【整体风格】【上装】【下装】【帽子】【饰品】【袜子】【鞋子】【眼镜】等 |
| 身份、职业、种族类 | 【职业】【身份】【种族】【校内职业身份】 |
| 语言、动作、表情类 | 【行为/习惯】【语言风格】【表情】【行为动作】【肢体语言】 |
| 嗜好、习惯类 | 【嗜好/习惯】【情结】 |
| 体质、疾病类 | 【体质】【疾病/残疾】【心理症状】 |
| 血缘关系、人际关系类 | 【家庭/家族】【学校/社会】【恋人相关】【人际组合】 |
| 武器、道具类 | 【日常用具】【武器】【文具】【劳动工具】【体育用品】【食物/饮品】等 |
| 复合型、综合类及其他类 | 详见参考.txt最后一节 |

#### 3.3 生成第一版文档

参照第一步学习的格式，生成第一版 md 文件：

- **萌属性一览表**：按大类→小类结构，每个属性附强度星级（★ 1-5）和出典
- **强度判定规则**：
  - ★★★★★ — 角色设定中反复明确强调，或为核心标签
  - ★★★★ — 有明确描述，表现突出
  - ★★★ — 有提及，表现明显
  - ★★ — 有暗示或轻度表现
  - ★ — 存在但非常边缘
- **出典**：优先使用用户原文中的描述；无原文则写"（暂无明确出典）"，**绝不凭空编造**
- **详细说明**：选取 2-5 个核心萌点展开，附原文引用（`>` 块引用格式）
- **语言风格**：客观叙事，避免主观评价；以事实描述为主

---

### 第四步：参考.txt 核验——最终质量门禁

**目的**：确保文档中所有属性均来自萌娘百科的规范分类体系，不存在生造信息。

#### 4.1 建立参考.txt 属性索引

读取 `data/参考.txt`，建立完整的属性查找表：

1. 逐行扫描，识别所有 `【】` 中的小类标签
2. 收集每个小类标签下出现的所有属性名
3. 输出格式：属性名 → `【小类标签】`

**参考.txt 中各属性的归属以实际文件内容为准，不得臆断。**

#### 4.2 逐条核验第一版文档中的属性

对第一版文档「萌属性一览表」中出现的**每一个属性**，执行以下检查：

```
FOR each 属性 in 第一版文档:
    IF 属性 in 参考.txt的属性索引:
        PASS — 记录其【小类标签】
    ELSE:
        FAIL — 标记为「未在参考.txt中找到」
        处理方式:
            a. 尝试模糊匹配（如同义词、近义词）是否能对应到参考.txt条目
            b. 若能对应，修正属性名为参考.txt中的标准名称
            c. 若无法对应，删除该属性，并在文档中注明「（属性名）无法通过核验，已移除」
```

**常见模糊匹配示例**：

| 用户/初稿描述 | 参考.txt 标准名称 |
|---|---|
| 银白色头发 | 银发 |
| 异色双瞳 | 异色瞳 |
| 小萝莉 | 幼女 或 美少女（需结合年龄判断） |
| 毒舌傲娇 | 毒舌 + 傲娇（拆分为两个独立属性） |
| 黑长直 | 长直（黑长直复合型，归入【复合型】） |
| 小正太 | 正太 |
| 过膝长袜 | 过膝袜 |

#### 4.3 修正文档

根据核验结果修正文档：

- **删除项**：所有无法在参考.txt中找到且无法模糊匹配的生造属性
- **修正项**：将非标准名称替换为参考.txt中的标准名称
- **新增项**：若参考.txt中存在用户描述有涉及但初稿遗漏的属性，可酌情补充
- **分类修正**：确保每个属性的【】小类标签与参考.txt一致

#### 4.4 终检确认

修正完成后，再次扫描文档，确认：

- [ ] 所有萌属性一览表中的属性都能在参考.txt中找到对应条目
- [ ] 每个属性的小类标签（【】）与参考.txt一致
- [ ] 所有出典均有用户原文支撑，无凭空编造
- [ ] 文档格式符合加里波.md的规范结构

若终检通过，输出最终版 md 文件。

## 核心数据结构

### 静态数据（`data/`）

| 文件 | 内容 |
|---|---|
| `attr_index.json` | 所有已知属性的列表（数千条），用于精确匹配 |
| `attr_desc.json` | 属性 → 萌娘百科词条描述的映射，**生成时作为参考，不得直接复制原文** |
| `hair_color_attr.json` | 发色属性枚举 |
| `eye_color_attr.json` | 瞳色属性枚举 |
| `male_attr.json` | 典型男性属性列表 |
| `female_attr.json` | 典型女性属性列表 |
| `参考.txt` | 萌娘百科属性分类体系，所有属性的分类以本文件为准 |
| `加里波.md` | 本项目的规范终稿格式，生成时严格参照 |

## 注意事项

1. **四步顺序不可跳过**：第一步学习格式、第二步读取用户信息、第三步初稿生成、第四步核验修正，每一步都是前一步的前提
2. **第四步是强制门禁**：任何在参考.txt中找不到的属性都不得出现在最终文档中，即使看起来"合理"
3. **歧义属性处理**：有些属性名有多种含义（如"天然呆"和"天然疯"），需结合用户描述上下文判断
4. **出典必须来自用户原文**：绝不能不提供出典；无法从用户描述中提取出典时，明确标注"（暂无明确出典）"
5. **中文编码**：所有 JSON 文件均使用 UTF-8 编码，key 和 value 均为中文字符串
6. **萌娘百科文章风格**：客观中立、以事实描述为主，避免过度主观评价
